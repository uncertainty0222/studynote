# HaxKai 트레이딩 어시스턴트 (`hax.mjs`)

Claude Code 대화 안에서 **라이브 시장 데이터**를 같이 보고 **켈리 비율에 맞는
포지션 크기**를 계산하기 위한 의존성 0 CLI. (Node 18+ 글로벌 `fetch`)

데이터 출처: Binance USDⓈ-M 선물 **공개 API** (API 키 불필요).

---

## ⚠ 선행 작업 — 네트워크 허용목록

`market` 커맨드는 `fapi.binance.com` 에 접속한다. 이 Claude Code 환경의
**네트워크 정책 허용목록에 `fapi.binance.com` 을 추가**해야 동작한다.
추가 전에는 친절한 안내 메시지와 함께 실패한다(크래시 아님).

- 설정: https://code.claude.com/docs/en/claude-code-on-the-web
- (선택) `api.binance.com` — 현물 가격 폴백용

`size` 커맨드는 순수 계산이라 **네트워크 없이도** 동작한다.

## 사용법

### 1) 시장 데이터 — `market`
```
node trading/hax.mjs market <SYMBOL> [--tf 4h,1d] [--limit 200]
```
심볼은 `TA`, `$TA`, `TAUSDT`, `TAUSDT.P` 모두 `TAUSDT` 로 정규화된다.

출력: 현재가·24h 변동·고저·거래량, 마크가·펀딩비, 그리고 타임프레임별로
ASCII 스파크라인, 레인지 내 위치, ATR(14), 근접 지지·저항(스윙 극값).

```
node trading/hax.mjs market TA --tf 1h,4h,1d
```

> ❗ HaxKai가 차트에 **직접 그린 손절·타겟 선**은 어떤 API에도 없다.
> 그 레벨은 시그널마다 텍스트로 전달하거나, 위 지지/저항 계산으로 대체한다.

### 2) 포지션 사이징 + 켈리 — `size` (네트워크 불필요)
```
node trading/hax.mjs size --entry <가격> --sl <가격> --risk <$> \
     [--tp1 .. --tp2 .. --tp3 .. --tp4 .. --p <승률> --side long|short --seed 5000]
```
출력:
- 손절폭, **수량 = risk ÷ 손절폭**, 명목가치, 시드 대비 리스크%(>2% 경고)
- 레버리지 표(2/3/5/7/10x): 증거금, 추정 청산가, **청산 < SL 경고**
- TP별 R배수 (TP1 < +1R 경고)
- `--p` 입력 시 켈리 `f* = p − (1−p)/R` (TP1 손익비 이항 근사), 하프·쿼터,
  음수면 "베팅 금지", 손익분기 승률

예 (대화 기록의 HUMA 셋업 재현):
```
node trading/hax.mjs size --entry 0.0245 --sl 0.021481 --risk 250 \
     --tp1 0.027001 --tp2 0.030007 --tp3 0.033012
# → 수량 ≈ 82,809, 명목 ≈ $2,029, 3x 증거금 ≈ $676, TP1 +0.83R
```

## 핵심 규율 (대화에서 합의된 전제)
1. **물타기 금지** — 진입과 동시에 SL 등록, 손실은 −1R로 고정.
2. **고정하는 것은 진입 금액이 아니라 1R(리스크 금액).** 포지션은 `1R ÷ 손절폭`으로 역산.
3. **레버리지는 명목가치를 맞추는 수단**일 뿐 고정 대상이 아니다. 증거금 부족 시
   레버리지를 올리지 말고 수량(=리스크)을 줄여라.
4. 켈리의 `p`·`R`은 **감이 아닌 실측치**여야 한다. 깨끗한 표본 전에는 풀 켈리 대신
   시드의 1~2% 고정 리스크로 시작.
